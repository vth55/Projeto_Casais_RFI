## ⚡ Comandos "/" (O teu arsenal de Workflows)

Basta escreveres `/` seguido do comando para eu ativar o guião oficial.

### 📋 GESTÃO E STATUS
| Comando | O que faz? | Quando usar? |
|---------|------------|--------------|
| **`/status`** | Mostra o progresso atual. | Ver o que falta do roadmap. |
| **`/plan`** | Cria um plano detalhado. | Antes de começar uma feature nova. |
| **`/orchestrate`** | Ativa o modo multi-agente. | Problemas complexos que envolvem tudo. |

### 🛠️ CRIAÇÃO E MELHORIA
| Comando | O que faz? | Quando usar? |
|---------|------------|--------------|
| **`/create`** | Cria uma nova app/módulo. | Começar algo do zero. |
| **`/enhance`** | Melhora uma feature existente. | Adicionar funções a uma vista. |
| **`/ui-ux-pro-max`**| Foca 100% no Design Casais. | Criar ou polir interfaces (UI). |

### 🔍 AUDITORIA E DEBUG
| Comando | O que faz? | Quando usar? |
|---------|------------|--------------|
| **`/auditoria`** | Protocolo Arqueólogo Total. | Sincronizar docs com código (100%). |
| **`/debug`** | Caça erros sistematicamente. | Quando a app "rebenta" ou há bugs. |
| **`/test`** | Gera e corre testes. | Garantir que o código está sólido. |

### 🚀 OPERAÇÕES
| Comando | O que faz? | Quando usar? |

| **`/brainstorm`** | Debate ideias socráticas. | Estás indeciso sobre uma lógica. |
