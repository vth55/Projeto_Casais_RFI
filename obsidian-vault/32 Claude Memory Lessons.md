# 32 Claude Memory Lessons

## What Worked Poorly
- long-running or heavy memory hooks in the interactive path
- opaque memory behavior with too much hidden machinery
- Windows-native `claude-mem` path under Antigravity/Claude Code when hooks and MCP got too invasive

## What Worked Better
- lighter memory systems outside the critical session path
- strong project docs plus compact curated memory
- separating durable knowledge from raw session automation

## Current Intended Model
- project docs remain authoritative detail
- Obsidian vault becomes human-readable, cross-AI durable memory
- `ICM` remains useful as lightweight automatic operational memory

## Practical Rule
- if a memory system can block the editor/chat path, it is too invasive
- if a memory note is too long to skim, it is too dense
